Overview

Executive summary

Albadi operates a production-grade sales platform. Customers design non-woven bags in real-time 3D, receive factory-accurate ILS pricing, and download branded PDF quotes. Every submission creates or updates a CRM lead. Operators manage the full cycle — WhatsApp conversations, pipeline, factory quotes, bot draft approval, and personalized 3D designer links.

Product

3D Configurator — specification

Full-screen designer at /configurator — standalone, embedded on the customer website, or opened via WhatsApp session link.

Fabric & Color

44 swatches in a scroll rail (keen-slider). Live GLB material update, SKU copy, orbit/zoom on bag mesh.

Logo

PNG / JPG / SVG up to 8 MB. 3D decal with scale, position, rotation. Real-time preview on bag front.

Quote & PDF

Size presets + custom, quantity tiers, shipping method. Live ILS factory price. Branded PDF with design snapshot.

Toolbar exports

Reset view, auto-rotate, PNG snapshot, fullscreen, 360° MP4 video export for sales follow-up.

Operator → customer flow: Send 3D Designer from lead card creates a session token and WhatsApp CTA. Customer opens link, design is pre-associated with the lead. PDF submit updates the same record.

Sessions & persistence

  • CRM → customer: WhatsApp link with ?t=<token>, 14-day session, pre-filled contact fields.
  • Customer → CRM: PDF submit upserts lead by phone, saves design row, sets stage INTAKE, tags source website_configurator or crm_3d_link.
  • Saved designs: Visible on lead Overview tab — color, logo thumbnail, quote total, re-open link.

Automation

WhatsApp bot & pipeline

All WhatsApp I/O runs through the self-hosted bridge. Tags, pipeline state, and custom fields live in Neon Postgres — GHL mirrors shared fields via webhooks.

9-step questionnaire (new leads)

StepQuestionOutcome
1–2Opening + trust contextBot introduces Albadi, sets expectations (~2 min)
3Shipping methodExpress (~25d) or sea (~90d)
4Quantity1k / 3k / 5k / 10k or custom amount
5Bag size6 popular presets, extended list, or custom H×D×W
6HandlesWith or without
7LaminationPremium finish yes/no
8Logo colors1, 2, or 3 print colors
9Confirmation gateSummary review → auto-price + quote send → INTAKE

Pipeline stages

  • NULL Questionnaire in progress — first inbound, no quote yet
  • INTAKE Auto-quote sent — awaiting customer engagement
  • DISCAVERY Discovery / commitment call with salesperson
  • FACTORY_WAIT Non-standard spec — factory estimate in flight
  • CONSIDERATION Final quote in customer hands — negotiation
  • WON / LOST Terminal — payment confirmed or explicit refusal

Side stages (manual drag only): FUTURE_FOLLOW_UP, NO_RESPONSE_REENGAGE

Diagram 7.2 — Inbound message routing

flowchart TD
  IN["Inbound WhatsApp"] --> WH["Bridge webhook"]
  WH --> ST{"Pipeline stage?"}
  ST -->|NULL| Q9["9-step questionnaire"]
  Q9 --> AQ["Auto-price + send quote"]
  AQ --> INT["Move to INTAKE"]
  ST -->|INTAKE+| INTNT["Classify intent"]
  INTNT --> MM{"Money moment?"}

  DQ --> AP["Operator approves in Drafts"]
  AP --> SEND["Send via bridge"]

Money moments (negotiating, reject, spec change) generate a draft for operator approval when ENABLE_DRAFT_QUEUE=1.

How to use

End-to-end workflows

Two primary paths — self-service website visitors and operator-initiated 3D designer links. Both converge on the same CRM lead record.

Website visitor (self-service)

  1. Open albadi.ecobrotherss.com/configurator
  2. Pick color → upload logo → configure quote
  3. Enter contact → download PDF
  4. CRM upserts lead, saves design

Operator 3D link

  1. Sign in → open lead in Leads or Conversations
  2. Click Send 3D Designer → WhatsApp CTA
  3. Customer completes design + PDF
  4. Record updates, designs on lead card

Technical

Data model & integrations

Neon PostgreSQL is authoritative for bot-internal state. GoHighLevel owns shared contact fields — DB mirrors via webhooks with idempotent merge.

Core database entities

TablePurposeOwner
leadsContact, pipeline stage, q_state, quote totals, bot flagsDB (+ GHL sync for shared fields)
messagesWhatsApp thread — sender: lead / bot / eliDB only
configurator_designsSaved 3D sessions — color, logo, quote JSONDB only
bot_draftsPending operator-approved repliesDB only
bot_decision_logSupervisor classifications + reasoningDB only
factory_quote_requestsFactory estimate lifecycleDB only
bridge_eventsWebhook audit + dedupe by evt_idDB only
lead_tagsTag membership by Hebrew nameGHL → DB mirror

Integration matrix

SystemDirectionWhat syncs
WhatsApp BridgeInbound + outboundMessages, contact enrichment, HMAC webhook
GoHighLevelBidirectionalStage, tags, name, phone, notes, tasks, opp value
bag-quote-appEmbed onlyiframe → CRM configurator, no local persistence
Meta Lead AdsInboundGoogle Sheets → API import
Vercel CronScheduledHourly bot classifier via /api/bot/cron

Security notes

Webhook verification

  • Bridge: HMAC-SHA256, 5-min replay window
  • Bot APIs: Bearer BOT_SECRET
  • GHL: signed payload validation

Data handling

  • Phone stored E.164 only
  • Logo uploads client-side, not persisted raw
  • Env secrets on Vercel — never in client bundle
