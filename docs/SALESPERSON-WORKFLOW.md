## 1. הבעיה (מבוסס נתונים חיים, 2026-06-12)

העסק חי — 1,000 הודעות ב-7 ימים, לידים חדשים כל יום, לקוחות כותבים "אתה לא חזרת אליי". אבל הפייפליין לא משקף את המציאות, **כי callbacks נופלים בין הכיסאות:**

המערכת **כבר מנתחת כל שיחה ומחלצת את ה"מתי לחזור"** — מתוך 96 שיחות מנותחות, ב-65 יש `next_steps` מפורש. דוגמאות אמיתיות:

| לקוח | שלב | מה סוכם בשיחה |
|---|---|---|
| Itzik Avigdor | INTAKE | "סוכם על שיחה מחר בשעה 9:00" |
| אמנון גמליאל | DISCAVERY | "סוכם לחזור בעוד רבע שעה עד עשרים דקות" |
| Or Laufman | CONSIDERATION | "סוכם שיחזור אלי עוד שעתיים" |

**האקדח המעשן — לידים שאמרו "תחזור" וסומנו `LOST` כי אף אחד לא חזר:**
- **Itay Sabbagh** → LOST: "סוכם לחזור ללקוח בעוד שעתיים"
- **ישי תעיז** → LOST: "סוכם שהנציג יחזור ללקוח בעוד חצי שעה"
- **אהרן** → LOST: "סוכם על פגישה בעוד חצי שעה"

**המסקנה:** המידע נתפס — אבל נשאר כהערה שאף אחד לא קורא בזמן. אין מנגנון שהופך "תחזור עוד שעתיים" ל**פעולה עם שעה**.

### מדדי בעיה נוספים (123 לידים פעילים)
- **0 מתוך 123** הזדמנויות משויכות לבעלים → אין "הלידים שלי".
- **74 LOST (60%)**, מתוכם 72 בלי סיבת אובדן → הפייפליין הוא בית קברות.
- **0 WON** מתועד → המערכת לא יודעת מה הצליח.
- **106 משימות פתוחות, 105 באיחור** → משימות מערכת (`bot_paused` וכו') שאף אחד לא סוגר = רעש שמסתיר את האמיתיות.
- **`last_response_at` ריק אצל כל 123** למרות 731 הודעות נכנסות → אי אפשר למיין לפי "מי חם".
- **תיוג אחד בלבד** על כל המערכת → מערכת הדגלים לא בשימוש.

## 3. העיקרון: Tasks = מנוע, Opportunities = מפה

> **Follow-up הוא לולאה *בתוך* כל שלב, לא מעבר שלב.**
> ליד נשאר ב-DISCAVERY דרך 3-4 callbacks עד שיש *תוצאה*. לכן אי אפשר לנהל את היום לפי עמודות הקנבן — בתוך עמודה אחת לא רואים "למי מתקשרים ב-14:00".

| כלי GHL | תפקיד | מתי נכנסים |
|---|---|---|
| 📞 **Tasks — "היום שלי"** | **המסך הראשי.** מסונן: *משויך אליי + תאריך≤היום*, ממוין לפי שעה. שם + סיבה + כפתור חיוג. | כל היום. זה המסך שחיים בו. |
| 🗺️ **Opportunities (קנבן)** | המפה — "איפה כל עסקה עומדת". | פעם-פעמיים ביום. |
| 📋 **Smart List לפי שלב** | תצוגה שמורה "ההזדמנויות שלי לפי שלב" — גיבוי. | לעבוד עמודה שלמה. |
| 📅 **Calendar** | רק פגישות בזמן *קבוע*. | לא ל-callback זורם. |

⚠️ השדה `follow_up_date` הוא **תאריך בלבד, בלי שעה** — לכן callback עם שעה ("עוד שעתיים", "מחר ב-9:00") **חייב להיות Task**, לא השדה הזה.

## 4. המסך: "היום שלי"

תצוגת Tasks ב-GHL, שמורה כ-filter, ממוינת לפי שעה:

```
היום שלי — [שם איש המכירות]                         🔔 3 באיחור
─────────────────────────────────────────────────────────
⏰ 09:00  📞 Itzik Avigdor        INTAKE      "סוכם שיחה ב-9:00"   [חייג]
🔴 עבר    📞 Or Laufman           CONSIDER.   "חזרה — עוד שעתיים"  [חייג]
⏰ 11:30  📞 אמנון גמליאל          DISCAVERY   "חזרה אחרי 20 דק'"   [חייג]
⏰ 14:00  💎 Nir Davidson          CONSIDER.   "עסקה 55K — סגירה"   [חייג]
```

- כל שורה = **משימת GHL** עם `dueDate` (תאריך+שעה), `assignedTo` = איש המכירות.
- מסנן רק סוגי משימה אקשן: `callback` + `sales_call`. **משימות מערכת (`bot_paused`, `needs_eli`) מוסתרות.**
- לחיצה → כרטיס הלקוח עם כל ההיסטוריה (ניתוח השיחות, ההצעה, ההערות).

## 5. ה-cadence בתוך כל שלב (הנוהל לאיש המכירות)

**העיקרון:** הליד נשאר בשלב כל עוד יש תור callbacks פתוח. עובר שלב רק כשיש **תוצאה**.

| שלב | פולואפ ברירת-מחדל | יוצא מהשלב כש... |
|---|---|---|
| **INTAKE** (קיבל הצעה אוטומטית) | שיחה תוך יום. לא ענה → יום 1 / 3 / 5 | ענה → DISCAVERY · 3 בלי מענה → NO_RESPONSE_REENGAGE |
| **DISCAVERY** (שיחת בירור) | **כאן ה-callbacks הזורמים** — כל אחד לפי הזמן שסוכם | הסכים → FACTORY_WAIT / CONSIDERATION · סירב → LOST |
| **FACTORY_WAIT** (אלי מתמחר) | עוקב; 3+ ימים → משימה "פנה למפעל" | מחיר סופי מוכן → CONSIDERATION |
| **CONSIDERATION** (מו״מ/סגירה) | callbacks שסוכמו + 24h / 3 ימים / 7 ימים | שילם → WON · סירב → LOST |

## 6. המנגנון הטכני: callback → GHL Task

**התשתית כבר קיימת ועובדת — צריך להוסיף חוט אחד.**

### מה כבר יש (אומת 2026-06-12)
- `analyzeCall(transcript)` ([call-analysis.ts](../lib/autoresponder/call-analysis.ts)) מחזיר `CallAnalysis` עם `next_steps: string[]` + `follow_up_urgency`.
- `deriveDesiredTasks(snapshot)` ([derive.ts](../lib/ghl-tasks/derive.ts)) כבר יוצר משימות GHL (`📞 פעולה אחרונה לפני קר`, `💎 סגור עסקה גדולה`) עם `{title, dueAt}`.
- `reconcile.ts` ([reconcile.ts](../lib/ghl-tasks/reconcile.ts)) מסנכרן desired↔GHL בלי כפילויות (טבלת `ghl_lead_tasks`).
- `createContactTask(contactId, {title, dueDate, assignedTo})` — **תומך ב-`dueDate` ISO מלא (שעה!) וב-`assignedTo` (user id של איש המכירות).** ✅

### מה צריך להוסיף
1. **`analyzeCall` — שדה callback מובנה.** להוסיף ל-`CallAnalysis`:
   - `callback_at: string | null` — datetime מקומי. מזינים ל-LLM את זמן השיחה ("השיחה ב-2026-06-12 14:30") והוא ממיר "עוד שעתיים"→`16:30`, "מחר ב-9"→`2026-06-13 09:00`.
   - `callback_reason: string | null` — טקסט קצר למשימה.
   - קוד-side: clamp (חייב עתידי, עד 60 יום), ועיגול לשעות עבודה (§9).
2. **signal kind חדש `callback_scheduled`** ב-`derive.ts` — `dueAt = callback_at`, `title = "📞 חזרה: <reason>"`, ו-`assignedTo` = איש המכירות.
3. **`reconcile` יעביר `assignedTo`** ל-`createContactTask` (היום לא מועבר).
4. **חיווט:** stage 4 של [process-recordings](../app/api/bot/process-recordings/route.ts) ו-[elevenlabs/sync-calls](../app/api/elevenlabs/sync-calls/route.ts) — אחרי שמירת הניתוח, אם `callback_at` קיים → ליצור desired task ולהריץ reconcile.

### זרימה מלאה
```
שיחה מוקלטת → Whisper → analyzeCall (מחלץ callback_at)
   → derive: callback_scheduled task (dueDate+שעה, assignedTo=מכירות)
   → reconcile → createContactTask → 📞 מופיע ב"היום שלי" של איש המכירות
```

## 7. הקמת GHL (חד-פעמי)

1. **להוסיף את איש המכירות כ-user ב-GHL** → לתפוס את ה-`user_id` שלו (ל-`assignedTo`) → לשמור כ-env `GHL_SALESPERSON_USER_ID`.
2. **להוסיף ערך `lead_owner`** חדש (למשל `🧑‍💼 <שם>`) לצד `👨 Eli` / `🤖 Bot` ([mapping.ts:135](../integrations/ghl/mapping.ts)).
3. **לשייך את כל ההזדמנויות הפעילות** לאיש המכירות (owner) — *אני יכול דרך ה-API*.
4. **לבנות תצוגת "היום שלי"** — Tasks: משויך=מכירות · תאריך≤היום · ממוין לפי שעה.
5. **להסתיר משימות מערכת** — לסגור/לארכב את 105 משימות הרעש; "היום שלי" מציג רק `callback` + `sales_call`.

## 9. פתוח לאישור — צריך החלטה ממך לפני Phase 1

1. **מי איש המכירות?** שם + להוסיף אותו כ-user ב-GHL (אני צריך את ה-user id).
2. **🔴 באיזה קו הוא מתקשר?** אם מהנייד הפרטי — השיחות שלו **לא** מוקלטות/מנותחות, אז ה-callback האוטומטי עובד רק על שיחות נכנסות ו-ElevenLabs; את שאר ה-callbacks הוא יוצר ידנית (30 שניות). אם דרך מספר ה-GHL / ElevenLabs (03-3822538) — **הכל אוטומטי.** *(זו ההחלטה הכי משפיעה.)*
3. **callback אוטומטי — מלא או טיוטה?** המלצה: **מלא** (זו משימה פנימית, לא הודעה ללקוח → אין סיכון).
4. **זמני ברירת מחדל:** "מחר" בלי שעה → 9:00? "מאוחר יותר" בלי זמן → אותו יום אחה"צ או מחר בבוקר?
5. **חלון חיוג:** callback שנופל בלילה/שבת → לדחות לחלון העבודה הבא? (יש כבר לוגיקת שעות-שקטות בבוט.)
